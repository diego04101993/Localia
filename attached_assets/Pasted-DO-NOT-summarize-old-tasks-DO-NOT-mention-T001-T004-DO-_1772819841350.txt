PHASE 1 — MULTI-MONTH MEMBERSHIP PLANS
==================================================

PROBLEM:
Right now plans are effectively monthly only. We now need support for:
- 1 month
- 3 months
- 6 months
- 12 months
- custom number of months if needed

BUSINESS RULE:
- 1 month = 30 days exactly
- 3 months = 90 days
- 6 months = 180 days
- 12 months = 360 days
- If a plan says 36 classes for 3 months, that means 36 classes FOR THE FULL CYCLE, not per month.

REQUIRED CHANGES

A) Plan model
Add a new integer field to membership plans:
- cycleMonths (default 1)

Backfill:
- existing plans => cycleMonths = 1

B) Create/Edit plan UI
In the Memberships tab:
- Replace fixed “Mensual (30 días)” with a configurable cycle selector.
- Use a clean select with options:
  - Mensual (1 mes / 30 días)
  - Trimestral (3 meses / 90 días)
  - Semestral (6 meses / 180 días)
  - Anual (12 meses / 360 días)
  - Personalizado
- If “Personalizado”, show numeric input for months > 0

Labels:
- “Ciclo”
- “Clases incluidas en todo el ciclo”
- “Precio total del ciclo”

Add helper text under classes:
- “Estas clases aplican para todo el ciclo completo, no por mes.”

C) Membership assignment / renewal logic
When assigning a plan to a client:
- paidAt = now
- expiresAt = now + (cycleMonths * 30 days)
- totalClassesPerCycle = plan.classesIncluded
- remainingClasses = plan.classesIncluded (or null if unlimited)
- status stays compatible with current logic

When renewing:
- preserve same plan and same cycleMonths
- paidAt = now
- expiresAt = now + (cycleMonths * 30 days)
- reset classes for the new cycle if not unlimited

D) Plan cards / client profile
Everywhere a plan is shown, display:
- cycle label:
  - Mensual / Trimestral / Semestral / Anual
  - or “{N} meses”
- exact duration in days
- classes per cycle
- price total

In client profile show:
- “Pagó el”
- “Vence el”
- “Ciclo”
- “Clases restantes”
- “Días restantes”

E) Alerts compatibility
Existing alerts (expired / expiring / no classes) must continue working using the new expiresAt logic.
