OBJETIVO GENERAL
==================================================

Agregar registro público SOLO para CLIENTES desde la pantalla de inicio de sesión web.

NO es para sucursales.
NO es para branch admins.
NO es para super admin.

Solo clientes finales.

Cuando un cliente se registre desde la web, debe poder crear su cuenta con:
- nombre
- apellidos
- fecha de nacimiento
- sexo
- celular
- correo electrónico
- contraseña

Después de registrarse:
- podrá iniciar sesión en la app web del cliente
- su perfil quedará guardado en la tabla de users o donde corresponda en la arquitectura actual
- su rol debe ser CUSTOMER
- esa información debe servir como perfil global del cliente

==================================================

1. El registro público es SOLO para clientes
2. Las sucursales NO se registran desde ahí
3. Si un cliente ya fue creado antes por una sucursal y luego quiere entrar por su cuenta:
   - debe poder completar/activar su acceso sin duplicar usuarios
   - revisar si ya existe por email
4. Si un cliente entra a una sucursal y se vuelve miembro, sus datos deben seguir apareciendo en el dashboard de esa sucursal, como ya funciona hoy
5. Si ese mismo cliente luego se vuelve miembro de otra sucursal, sus datos también deben poder reutilizarse sin duplicar identidad
6. Debe mantenerse la lógica actual donde distintas sucursales pueden ver a sus propios clientes-miembros, pero el perfil base del cliente es el mismo usuario
7. NO crear duplicados innecesarios de usuario por cada sucursal
8. Mantener integridad de datos y compatibilidad con el modelo actual
