FASE 3 — Lógica real de “clases restantes / vencimiento / descuentos”
REQUISITOS DE NEGOCIO (muy importante):
- Cuando un cliente tiene plan asignado:
  - Debe existir:
    - membership_start_date
    - membership_end_date (si plan tiene days) o null si sin vencimiento
    - classes_total (si plan tiene classes) o null si ilimitado
    - classes_remaining (si classes_total existe) inicia igual a total
- Descuento de clase:
  - Si el cliente ASISTE a una reserva => se descuenta 1 clase (si no es ilimitado)
  - Si NO asiste y ya pasó la hora de inicio (o fin) => se descuenta 1 clase (si no es ilimitado)
  - Si CANCELA a tiempo => NO se descuenta
- “Cancela a tiempo”:
  - Crear una regla configurable por sucursal: cancel_cutoff_minutes (default 120 min).
  - Si cancela antes de cutoff => no descuento
  - Si cancela tarde => descuento 1 (si no es ilimitado)
- Congelado/inactivo:
  - frozen: no puede reservar y no puede marcar asistencia
  - inactive: no puede reservar (pero historial visible)

IMPLEMENTACIÓN (sin romper):
A) DB/Migración:
- Agregar fields necesarios en memberships o tabla equivalente:
  - membership_start_date, membership_end_date
  - classes_total, classes_remaining
  - cancel_cutoff_minutes (en branches/settings) o en branches
- Mantener compat con datos actuales (defaults null).

B) Backend:
- Al asignar plan a cliente:
  - set start_date=now
  - calcular end_date = now + days (si days)
  - classes_total = plan.classes (si existe)
  - classes_remaining = plan.classes (si existe)
- En endpoint de marcar asistencia / actualizar estado de reserva:
  - Si status pasa a “attended” => aplicar descuento
  - Si status pasa a “no_show” o se marca como “no asistió” => aplicar descuento
- En cancelación:
  - Validar cutoff con fecha/hora clase
  - Guardar si fue “canceled_on_time” (bool) y si “late_cancel” (bool)
  - Si late_cancel => aplicar descuento

C) Jobs / Auto-descuento por tiempo:
- Cuando una clase ya pasó y el cliente estaba reservado y no se marcó asistencia:
  - marcar como “no_show” automáticamente (solo si pasó el tiempo fin)
  - aplicar descuento
- Implementación simple:
  - Al cargar Reservas del día/semana, correr una rutina “reconcilePastBookings” que revise bookings pasados no resueltos y los cierre. (Evitar cron si no existe).
  - Debe ser idempotente (no descontar doble).

D) UI:
- En perfil del cliente:
  - Mostrar “Clases restantes” y “Vence: dd/mmm/aaaa” (o “Sin vencimiento”).
  - Si classes_remaining = 0 => badge rojo “Sin clases” + CTA “Asignar plan”.
- En dashboard (Resumen):
  - Agregar bloque de notificaciones/alertas:
    - “Clientes sin clases: N”
